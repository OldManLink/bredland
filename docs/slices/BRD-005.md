# BRD-005 — Implement NOC Dashboard

## Status

Complete

## Goal

Implement the first iteration of a mobile-friendly NOC dashboard for visualising the current state of telemetry clients.

This slice establishes the dashboard deployment workflow and a simple framework for adding future status cards.

---

## Design

### Dashboard deployment

Introduce a deployment target for the NOC dashboard using the existing template rendering and deployment infrastructure.

The initial vertical slice should render a minimal dashboard page, deploy it to Oderland, verify the deployment, and open the live page in the default browser.

Deployment scripts should use shared helper functions where practical to minimise duplicated SSH logic.

---

### Dashboard framework

Implement a lightweight dashboard page suitable for both mobile and desktop browsers.

Initially display the most recent telemetry record from each heartbeat stream to demonstrate end-to-end data retrieval.

This establishes the foundation for future card-based status displays without yet interpreting the telemetry.

---

### Status cards

Introduce the first two dashboard cards:

* MikroTik
* Bredland

Initially the cards may display only the latest telemetry record.

The design should allow each card to evolve independently, eventually becoming responsible for:

* interpreting its own telemetry
* determining its own health status
* presenting appropriate operator actions
* exposing additional detail when expanded

The dashboard itself should remain responsible only for arranging cards and presenting overall page structure.

---

## Acceptance Criteria

* A dashboard deployment script successfully deploys the dashboard to Oderland. ✅
* The deployed page opens automatically in the default browser. ✅
* The dashboard displays the latest telemetry record for both MikroTik and Bredland. ✅
* The dashboard layout is suitable for mobile devices while remaining comfortable on desktop browsers. ✅
* The design establishes a framework for future card-based expansion without requiring changes to existing cards. ✅
