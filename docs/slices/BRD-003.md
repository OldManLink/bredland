# BRD-003 — Generic Telemetry Endpoint

## Status

Complete

## Goal

Generalise the current MikroTik-specific heartbeat endpoint into a generic telemetry endpoint capable of receiving telemetry from multiple hosts.

This slice preserves uninterrupted heartbeat collection throughout the migration.

---

## Design

### Generic endpoint

Rename the deployed endpoint from:

```
mikrotik.php
```

to:

```
telemetry.php
```

Rename the associated configuration file accordingly.

Deployment scripts and templates shall use the new names.

---

### RouterOS client

Rename the RouterOS objects to distinguish between the script and its scheduler.

```
Script:
    telemetry-heartbeat

Scheduler:
    telemetry-heartbeat-5m
```

The deployment shall be idempotent.

The legacy `noc-heartbeat` objects shall be retired only after the new deployment has been verified.

---

### Host-specific authentication

Replace the single shared token with host-specific authentication.

For this slice, configure a single host:

```
mikrotik
```

using its own authentication token.

The endpoint shall validate the supplied host against the configured token before accepting telemetry.

---

Generic telemetry payload

Reduce the number of fixed request parameters.

The client shall always send:

* host
* token
* uptime
* fields

where fields is a comma-separated list naming the additional telemetry parameters included in the request.

For example:

host=mikrotik
token=...
uptime=3d14:10:06
fields=version,model,cpu_load,free_memory,total_memory
version=7.23.1 (stable)
model=RB4011iGS
cpu_load=1
free_memory=882528256
total_memory=1073741824

The endpoint shall always record the mandatory fields together with the server-generated timestamp and remote IP address.

For each name listed in fields, the endpoint shall copy the corresponding request parameter into the stored JSON record.

---

## Acceptance Criteria

* Generic telemetry endpoint deployed. ✅
* RouterOS client renamed and deployed. ✅
* Existing heartbeat cadence preserved. ✅
* Legacy RouterOS objects retired. ✅
* Host-specific authentication implemented. ✅
* Generic `fields` payload implemented. ✅
* Heartbeat continuity maintained throughout the migration. ✅

---

## Notes

The migration intentionally keeps telemetry flowing throughout.

Where possible, endpoint changes are deployed before client changes, allowing the new endpoint to accept both old and new request formats until the migration is complete.
