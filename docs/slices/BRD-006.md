# BRD-006 — Improve NOC Mobile Experience

## Status

Planned

## Goal

Improve the day-to-day usability of the NOC when used as a home-screen web application on iPhone.

This slice focuses exclusively on operator experience, introducing lightweight interactions that reduce visual clutter while making the dashboard feel more like a native application.

---

## Design

### Collapsible telemetry

Hide the raw JSONL telemetry beneath each status card by default.

Each card should expose a small drawer handle (`=`) allowing the operator to reveal the raw telemetry on demand.

The implementation should use standard HTML where practical and avoid unnecessary JavaScript.

---

### Pull-to-refresh

Implement pull-to-refresh for iOS home-screen web applications.

Since iOS PWAs do not currently support native pull-to-refresh, provide a lightweight JavaScript implementation that reloads the dashboard when the user performs a downward pull gesture at the top of the page.

The implementation should remain visually simple and should not interfere with normal page scrolling.

---

## Acceptance Criteria

* Raw telemetry is hidden by default. ☐
* Each card can be expanded and collapsed independently. ☐
* The drawer handle remains unobtrusive while clearly indicating additional detail is available. ☐
* Pull-to-refresh works reliably when the dashboard is installed as an iPhone home-screen web app. ☐
* Normal scrolling behaviour is unaffected. ☐
* The implementation introduces only minimal, self-contained JavaScript. ☐
