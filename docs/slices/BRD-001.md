# BRD-001 – MikroTik NOC Heartbeat

## Status

Complete

## Goal

Create a lightweight heartbeat from the MikroTik router to the NOC.

The heartbeat should periodically POST basic health information to an external endpoint.

The solution must be secure, idempotent and easy to redeploy.

## Scope

This slice includes:

* RouterOS heartbeat template.
* Template renderer.
* Deployment script.
* NOC endpoint.
* Verification that heartbeat records are appended to a JSONL file.

## Design

### Router

The MikroTik stores a deployment-specific endpoint URL and authentication token.

Both values are injected during deployment and are **never committed** to version control.

The RouterOS scheduler periodically executes the heartbeat script.

### Deployment

The repository contains only templates.

A deployment script:

1. reads deployment-specific secrets;
2. renders the RouterOS template;
3. uploads it to the router;
4. installs or updates the RouterOS scheduler.

Deployment must be idempotent.

Running the deployment multiple times must not create duplicate scripts or scheduler entries.

### NOC

The endpoint validates:

* endpoint-specific authentication token;
* required request parameters.

Successful requests append one JSON object per line to:

```text
<secret-data-directory>/mikrotik.jsonl
```

The endpoint performs no retries.

The endpoint filename and storage location are deployment-specific and are injected outside version control.

## Data

Initial heartbeat fields:

* hostname
* uptime
* CPU load
* total memory
* free memory

Additional fields may be added in future slices.

## Security

This slice follows the repository security policy.

The repository must remain publishable at any time.

Accordingly:

* authentication tokens are not committed;
* endpoint URLs are not committed;
* storage paths are not committed;
* deployment-specific configuration is injected outside version control.

## Acceptance Criteria

BRD-001 is complete when:

* RouterOS heartbeat template exists. ✅
* Renderer exists. ✅
* Deployment script exists. ✅
* Deployment is idempotent. ✅
* Router scheduler is installed automatically. ✅
* `<secret-mikrotik-endpoint>` accepts valid requests. ✅
* Invalid authentication is rejected. ✅
* The JSONL file is created automatically if required. ✅
* A new JSON object is appended every configured interval. ✅
* No deployment-specific information exists anywhere in the repository. ✅
