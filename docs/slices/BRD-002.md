# BRD-002 — Daily NOC Log Compression

## Status

Complete

## Goal

Compress completed daily NOC log files on Oderland.

The current day's log must remain uncompressed so that heartbeats can continue to append normally.

Example:

```
mikrotik-2026-06-25.jsonl
→
mikrotik-2026-06-25.jsonl.gz
```

This prepares the data directory for long-term retention while keeping the active log simple.

---

## Design

### Compression script

Create a rendered shell script on Oderland:

```
compress-noc-logs.sh
```

The script shall:

* operate on `NOC_DATA_DIR`
* determine the current UTC date using `date -u`
* compress every `*.jsonl` file except today's active file
* skip files already compressed
* leave today's file untouched

The script should use:

```
gzip -n
```

to produce deterministic output.

---

### Scheduling

Install a daily cron job on Oderland:

```
10 2 * * * /path/to/compress-noc-logs.sh
```

Cron runs in the server's local timezone (CEST/CET).

The script itself always uses UTC when determining today's filename, ensuring correct behaviour across daylight-saving transitions.

---

### Deployment

Provide a deployment script:

```
scripts/deploy-oderland-log-rotation.sh
```

Responsibilities:

* load secrets
* render the compression script
* upload to Oderland
* make executable
* install/update a managed crontab entry
* verify installation

Deployment shall be idempotent.

---

### Templates

Add:

```
oderland/compress-noc-logs.sh.template
```

The template shall use placeholders rather than hard-coded paths.

---

## Acceptance Criteria

* Yesterday's completed log files are compressed. ✅
* Today's active log remains uncompressed. ✅
* Running the script repeatedly is safe. ✅
* Deployment is idempotent. ✅
* Cron entry is managed automatically. ✅
* Existing heartbeat functionality is unaffected. ✅

---

## Notes

The compression script intentionally compresses every completed `.jsonl` file except today's active file.

This means the system automatically catches up if the cron job fails to run for one or more days, without requiring any special recovery logic.
