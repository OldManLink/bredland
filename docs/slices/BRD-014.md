# BRD-014 — Linux shell test framework

## Status

Planned

## Goal

Run all shell script tests in a Linux environment, both locally and in GitHub Actions.

Shell scripts in the Bredland project are developed on macOS but deployed exclusively to Linux systems (Oderland and Raspberry Pi OS). Testing them directly on macOS introduces unnecessary BSD/GNU compatibility issues and reduces confidence that successful local tests accurately reflect production behaviour.

This slice establishes a common Linux-based test environment for all shell scripts.

---

## Design

### Linux test environment

Shell script tests shall execute inside a Linux Docker container.

The same test runner shall be used both locally and in GitHub Actions to ensure identical behaviour across development and continuous integration.

Docker shall be considered a required development dependency. If Docker is unavailable, the shell test suite shall fail immediately rather than silently skipping tests.

The framework shall be general-purpose so that future shell scripts can be added to the test suite without additional infrastructure work.

---

## Acceptance Criteria

* All shell script tests execute inside a Linux Docker container. ✅
* Local development and GitHub Actions use the same shell test runner. ⬜
* Shell tests fail if Docker is unavailable. ✅
* Existing shell script tests pass without modification in the Linux environment. ✅
* Future shell script tests can be added without additional infrastructure changes. ✅
