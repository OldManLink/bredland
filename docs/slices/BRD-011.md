# BRD-011 — Desktop Automatic Refresh

## Status

Planned

## Goal

Restore automatic dashboard refresh for desktop operators without disrupting the mobile experience.

The introduction of pull-to-refresh in BRD-006 made automatic refresh unnecessary—and occasionally frustrating—when using the dashboard as an installed iPhone web application. Desktop use, however, benefits from periodically refreshing the displayed telemetry without operator intervention.

---

## Design

### Desktop-only refresh

Automatically reload the dashboard at a fixed interval when viewed on desktop devices.

The refresh should be enabled only for desktop layouts, leaving mobile devices entirely under manual control via pull-to-refresh.

The implementation should be lightweight and require no server-side changes.

---

## Acceptance Criteria

* The dashboard automatically refreshes at a configurable interval on desktop. ⬜
* Mobile devices no longer perform automatic refreshes. ⬜
* Pull-to-refresh remains the primary refresh mechanism on mobile. ⬜
* The implementation is lightweight and self-contained. ⬜
