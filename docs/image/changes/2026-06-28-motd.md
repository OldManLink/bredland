# Static SSH MOTD banner

**Date:** 2026-06-28

## Motivation

Give Bredland a clear identity when logging in via SSH.

The implementation deliberately uses Debian's `/etc/update-motd.d/` mechanism so that future changes can introduce dynamic status information without changing the login mechanism.

The initial implementation is intentionally static.

## Procedure

Create:

```
/etc/update-motd.d/99-bredland
```

The file shall:

* be a POSIX shell script;
* have executable permissions (`chmod +x`);
* write a static ASCII-art banner to standard output.

The banner identifies the machine as:

* **Bredland**
* **Home Infrastructure Node**

and includes the subtitle:

```
Telemetry • Monitoring • Automation
```

No dynamic information is displayed by this version.

## Verification

1. Verify the script is executable.
2. Open a new SSH session:

```
ssh bredland
```

3. Confirm that:

* the Bredland banner is displayed before the login prompt;
* the standard Debian MOTD continues to appear after the custom banner.

## Rollback

Remove:

```
/etc/update-motd.d/99-bredland
```

or remove its executable permission.

## Notes

This change is intentionally implemented using the Debian MOTD framework rather than modifying system login files.

Future changes may replace the static output with dynamically generated status information (for example uptime, temperature, heartbeat status, or telemetry summaries) without changing the installation procedure.
