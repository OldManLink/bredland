# Volatile systemd journal

**Date:** 2026-06-29

## Motivation

Ensure that Bredland's system journal is stored in RAM rather than persistently on the SD card.

Bredland will run periodic systemd services such as the heartbeat client. Those services may write regular status or error output to the journal.

Keeping the journal volatile reduces unnecessary SD card writes and helps preserve the life of the card.

## Procedure

Create:

```
/etc/systemd/journald.conf.d/volatile.conf
```

The file shall configure journald to use volatile storage only:

```
[Journal]
Storage=volatile
```

Remove the persistent journal directory if it exists:

```
sudo rm -rf /var/log/journal
```

Restart systemd-journald:

```
sudo systemctl restart systemd-journald
```

## Verification

Verify the configured storage mode:

```
grep -E '^\s*Storage=' /etc/systemd/journald.conf /etc/systemd/journald.conf.d/*.conf 2>/dev/null || true
```

Expected result:

```
/etc/systemd/journald.conf.d/volatile.conf:Storage=volatile
```

Verify journal storage locations:

```
ls -ld /var/log/journal /run/log/journal 2>/dev/null
```

Expected result:

* `/run/log/journal` exists;
* `/var/log/journal` does not exist.

Check current journal disk usage:

```
journalctl --disk-usage
```

The output should refer only to volatile journal storage.

## Rollback

Remove:

```
/etc/systemd/journald.conf.d/volatile.conf
```

Optionally recreate the persistent journal directory:

```
sudo mkdir -p /var/log/journal
sudo systemctl restart systemd-journald
```

## Notes

This change is intentionally made before installing the Bredland heartbeat timer.

The heartbeat service is expected to run every five minutes. Its logs should therefore follow the system-wide RAM-backed journal policy rather than creating persistent SD card writes.
